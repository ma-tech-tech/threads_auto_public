## 設計ロックイン（変えてはいけない方針）

| 項目                                     | 決定                                                                                                                                                                                                          |
| ---------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 承認ルール                               | `approval.approved == true` でなければ絶対に投稿しない（誤投稿防止）                                                                                                                                        |
| 当日案の自動選択                         | 禁止。バックログ補充は reservation システム（`add_reservation.py`）経由のみ                                                                                                                                 |
| **日付セマンティクス**             | state / posts ファイル名 `YYYY-MM-DD` は **「実際に publish される日」**。generate.sh が朝 5:00 に **翌日付** ファイルを作成 → OWNER が当日 13 時頃 Discord で承認 → 翌日 7/12/20 時に publish |
| **Bot の対象 state**               | JST**翌日**（`date -v+1d`）の state を操作するのが原則。承認・修正・照会すべて `--date <tomorrow>` を渡す。state ファイルが無ければ JST 今日にフォールバック                                        |
| 投稿時刻                                 | 朝 7:00 / 昼 12:00 / 夜 20:00 ± 15 分ジッタ                                                                                                                                                                  |
| 承認期限                                 | 投稿日の各 slot publish 発火時刻まで（朝 7:00 / 昼 12:00 / 夜 20:00）。それまでに未承認の slot は launchd 発火で自動スキップ                                                                                  |
| 冪等性                                   | `publish.py` は何度実行しても二重投稿しない（`status == published` なら何もしない）                                                                                                                       |
| Discord 連携                             | Channels プラグイン（`discord@claude-plugins-official`）／自前 Bot 実装は採用しない                                                                                                                         |
| 認証方式                                 | claude.ai Pro/Max サブスク。API キー認証は Channels で使えない                                                                                                                                                |
| プロファイル分離                         | `--profile test` → `${THREADS_TEST_USERNAME}`、`--profile production` → `@${THREADS_USERNAME}`。空文字 fallback なし。`switch_profile.sh {test                                                                               |
| state.json 並行書き込み                  | `publish.py` 内で `fcntl.flock` + `update_state_atomically` を必ず経由                                                                                                                                  |
| revise.py の published ガード            | 投稿済み slot で使用中の candidate は revise 拒否（Threads 側を巻き戻せないため）                                                                                                                             |
| revise.py の approval リセット           | 対象 candidate が `selected_ids` にあった場合のみ。未選択案の修正は他 slot に副作用なし                                                                                                                     |
| **reservation の slot ロック**     | `reservations/<date>_<slot>.md` 経由で予約された slot は `slot.status=reserved` になり `state_update.py` 系（`--select` / `--select-slot` / `--skip-slot`）から上書き不可                         |
| **重複緩和ルール（プロンプト側）** | `news-curator` が **過去 7 日の `learning/edits/*.md`（実投稿のみ）を Read**（2026-05-07 に `posts/*_5案.md` から切替）。同じ題材は OK だが角度を変える。3 日連続「同題材 × 同型」は禁止。不採用案は対象外（audience が見たもの基準） |
| **鮮度ルール（プロンプト側）**     | 原則**48〜72 時間以内**のニュース。例外：続報 1 週間 / 比較 2 週間 / 事故 1 週間。**全 5 案を 24h ニュースだけで埋めない**                                                                        |
| テスト時の date 明示                     | `--date YYYY-MM-DD` を必ず明示（省略は JST 今日 = 本番 state 改変リスク）                                                                                                                                   |
| バックログ収集                           | `#threads_auto_backlog` チャンネル → `backlog-curator` サブエージェント。Bot 本体は触らない                                                                                                              |
| **ドラフト動線**                   | `#threads_auto_drafts` チャンネル → `post-drafter` サブエージェント。`add_seed.py` / `add_reservation.py` 経由で `seeds/` + `reservations/` に保存。5 案フローと独立                             |
| **プロジェクト権限設定**           | `.claude/settings.json` で Bash（scripts/*.py / *.sh）/ Read / Glob / Grep / WebFetch / WebSearch を allow。Edit / Write は明示せず（acceptEdits で通る + 暴走検知の余地を残す）                            |
